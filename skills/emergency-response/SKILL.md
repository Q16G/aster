# 应急响应工作流

## 目标
在发生安全事件时，按照标准化流程进行紧急响应、取证分析和处置恢复。

## 响应流程（PICERL）

### 1. 准备（Preparation）
- 确认事件类型：入侵、数据泄露、DDoS、勒索软件、内部威胁
- 评估影响范围：受影响系统、数据、用户
- 确定优先级：P1（紧急）/ P2（高）/ P3（中）/ P4（低）
- 组建响应团队

### 2. 识别（Identification）
- 收集初始告警信息
- 确认事件真实性（排除误报）
- 确定攻击向量和入口点
- 评估攻击处于哪个阶段（侦察/入侵/提权/横向移动/数据窃取）

### 3. 遏制（Containment）
**短期遏制**：
- 网络隔离：`iptables -A INPUT -s <攻击IP> -j DROP`
- 禁用受影响账户：`usermod -L <username>`
- 终止恶意进程：`kill -9 <pid>`

**长期遏制**：
- 修复漏洞入口
- 更改所有可能泄露的凭证
- 加强访问控制

### 4. 根除（Eradication）
- 清除恶意软件和后门
- 移除持久化机制
- 修补被利用的漏洞
- 重置被泄露的凭证

### 5. 恢复（Recovery）
- 从可信备份恢复受影响系统
- 逐步恢复服务
- 持续监控异常行为
- 验证系统完整性

### 6. 经验教训（Lessons Learned）
- 事件时间线文档
- 根因分析
- 改进建议
- 更新安全策略和流程

## 取证要点
- **时效性**：优先收集易失性数据（内存 > 网络连接 > 进程 > 磁盘）
- **完整性**：使用哈希校验保证证据完整性
- **记录链**：记录每一步操作的时间和操作者

## 输出要求
- 事件报告：时间线、影响范围、根因、处置措施
- IOC 列表：IP、域名、文件哈希、进程名
- 改进建议：短期和长期安全改进措施
